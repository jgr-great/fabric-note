MSP身份有效性规则¶

如MSP说明中所述，MSP可以配置有一组根证书颁发机构（rCA），以及可选的一组中间证书颁发机构（iCA）。MSP的iCA证书必须完全由MSP的rCA或iCA之一签名。MSP的配置可能包含证书吊销列表或CRL。如果CRL中列出了MSP的任何根证书颁发机构，则MSP的配置中不得包含CRL中也包含的任何iCA，否则MSP设置将失败。

每个rCA都是认证树的根。也就是说，每个rCA可能是一个或多个iCA证书的签名者，而这些iCA将是其他iCA或用户证书的签名者。这里有一些例子：

        rCA1                rCA2         rCA3
      /    \                 |            |
   iCA1    iCA2             iCA3          id
    / \      |               |
iCA11 iCA12 id              id
 |
id

默认的MPS实施接受由相应机构签名的X.509证书作为有效身份。在上图中，只有由iCA11，iCA12，iCA2，iCA3和rCA3签名的证书才被视为有效。内部节点签名的证书将被拒绝。

请注意，如果在MSP配置中指定了一个或多个组织单位，则证书的有效性也会以类似的方式受到影响。回想一下，在MSP配置中将组织单位指定为两个值对，例如分别表示代表该组织单位的证书颁发机构的（父证书，ou-string）和实际组织单位标识符。如果证书C由在MSP配置中指定了组织单位的iCA或rCA签名，则C除其他要求外，如果在其OU字段中包括ou-string，则认为C有效。
